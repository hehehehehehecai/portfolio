🎬 个人项目：平行信箱 (Parallel Mailbox)

📌 项目简介
本项目是一款主打“跨时空情感对话”的跨平台移动端应用。它打破了传统 AI 聊天机器人枯燥的问答模式，开创了极具叙事感的“平行宇宙通信”工作流。
作为一名敏锐观察现代人情感焦虑的开发者，我主导开发了这款工具。用户可以向“平行宇宙的自己”倾诉现实中的遗憾与焦虑，智能体不仅能自主提取用户的真实性格底色与现实困境，还能实时生成带有深度共情、极具代入感的专属第一人称回信。

核心价值： 本项目完成了从 0 到 1 的独立设计与全栈闭环开发，不仅是对大语言模型 (LLM) 在心理陪伴领域落地场景的深度挖掘，更展现了如何通过细腻的 UI/UX 设计与严谨的 Serverless 架构，将一项极客技术转化为有温度的商业级产品。

💡 设计思路与反思 (Product Philosophy)
1. 情绪价值优先于纯粹工具（产品调性设计）： 抛弃传统 App 枯燥的排版，全局采用“深空蓝”渐变主题与半透明毛玻璃卡片（Glassmorphism）。引入“伪实体化”的交互仪式感——为每一封回信动态生成专属的“宇宙坐标”与倾斜的红色虚拟邮戳，配合打字机动效与手机物理震动（Haptics），将产品的情绪价值拉满。
2. 千人千面的动态共情（AI PM 思维）： 纯粹的通用 AI 回复显得敷衍。我设计了“时空档案”系统，多维采集用户的 MBTI 性格与现实职业现状。在前端动态拼接高优 System Prompt，使大模型能够根据用户遗憾“反转”身份（如让 ENTJ 给出霸气开导，让 INFP 给予温柔共情），在“智能”与“代入感”之间找到最佳平衡。
3. 克制与护城河边界（商业风控视角）： 考虑到真实商业运行的 ROI 与 App Store 的严格审核，系统不仅内置了“每日寄信限流”机制（控制 Token 成本的同时制造时空信道的“稀缺感”），还严格接入了免责声明、EULA 弹窗与账号彻底注销机制，提前化解潜在的合规风险。

🚀 核心功能
🕸️ 高维 Prompt 身份注入引擎： 突破传统单轮对话的苍白感。系统在发送信件前自动获取用户“时空档案”，将现实世界的痛点动态转换为平行宇宙的“反转人设”，极大地提升了 LLM 角色扮演的逼真度。
🎛️ 电影级沉浸式 UI/UX 设计： 彻底重构前端组件，实现带有时空坐标号（如 Universe-897）的红色倾斜印章效果。结合定制的打字机组件（Typewriter）展开信件，让文字在屏幕上拥有呼吸感。
🛡️ 全栈数据流与强隐私保护： 深度集成 Supabase Auth，利用 PostgreSQL 行级安全策略 (RLS) 实现严格的数据权限隔离。即便是在公开网络环境中，也确保了每一封私密情感信件的绝对安全与云端双向同步。
⏱️ 商业级防御逻辑： 具备完整的工业级兜底设计。包含基于 AsyncStorage 的每日 3 封投递限流、针对弱网环境的 15 秒强制超时中断（AbortController），以及面对空数据库查询的平滑回落机制。

⚔️ 技术亮点与挑战 (Challenges & Solutions)
挑战一：跨端 Flexbox 布局塌陷与响应式降级
问题： 在 Web 端预览时，原本应该铺满全屏的星空背景经常“缩水”成狭窄的竖条，或者复杂的导航栏元素严重重叠，导致 UI 彻底崩坏。
方案： 深度剥离视图层级，对全局 App.tsx 根节点实施严谨的布局重构。确立“外层容器负责 100% 铺满、内部容器负责 maxWidth 居中约束”的规范，彻底解决了 React Native Web 下经典的布局适配难题。

挑战二：异步数据流中的“静默崩溃”与黑盒报错
问题： 早期版本在游客身份投递信件时按钮“毫无反应”，由于代码对空结果处理不当导致后续 AI 渲染管线彻底中断。
方案： 将死板的 .single() 查询安全替换为 .maybeSingle()，并为发送信件全链路植入极强的 Try-Catch 错误捕获网。通过层层控制台日志记录与状态重置保障，消灭了所有导致死锁的隐性 bug。

挑战三：跨平台底层 API 的兼容性陷阱
问题： 在开发注销账号与删除信件功能时，带有双按钮的 Alert.alert 在 Web 端被浏览器默认拦截导致逻辑失联。
方案： 引入 Platform.OS === 'web' 进行环境嗅探，在 Web 端优雅降级调用原生的 window.confirm，移动端保留原生 UI，确保了多端交互的绝对一致性。

📝 结语
"Parallel Mailbox" 的开发是一场从“敲击代码”到“雕琢产品”的蜕变之旅。它让我跳出了单纯追求功能堆砌的程序员思维，转而以全栈工程师兼产品经理的视角去审视整个应用流。大模型的人格化输出、边缘数据的安全兜底、以及用户在等待回信时的情绪反馈，共同构成了产品的核心竞争力。未来，我希望能将这种对用户体验的极致追求带入到更多 AI 落地场景中，创造出真正触动人心的技术产品。
